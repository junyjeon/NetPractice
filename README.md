## 🎯 프로젝트 소개
NetPractice는 TCP/IP 주소 지정과 서브넷의 기본 개념을 학습하는 시스템 관리 프로젝트입니다. 웹 인터페이스를 통해 다양한 네트워크 구성 문제를 해결하며 실제 네트워크 설정 방법을 배웁니다.

### 주요 학습 목표
- IP 주소 체계 이해
- 서브넷 마스크 계산
- 네트워크/호스트 주소 구분
- 기본적인 네트워크 구성
- TCP/IP 프로토콜의 기초

## 📡 IP 주소

### 1. IP 주소 클래스
```
클래스 A: 255.0.0.0     (1-127)
클래스 B: 255.255.0.0   (128-191)
클래스 C: 255.255.255.0 (192-223)
```

### 2. 특수 목적 IP
```
- 루프백: 127.0.0.0 ~ 127.255.255.255
- 사설 IP:
  • 10.0.0.0 ~ 10.255.255.255 (10.0.0.0/8)
  • 172.16.0.0 ~ 172.31.255.255 (172.16.0.0/12)
  • 192.168.0.0 ~ 192.168.255.255 (192.168.0.0/16)
```

### 3. 예약된 주소
```
- 네트워크 주소: 호스트 부분이 모두 0
- 브로드캐스트 주소: 호스트 부분이 모두 1
- 예시:
  • 192.168.0.0   (네트워크 주소)
  • 192.168.0.255 (브로드캐스트 주소)
```

## 🎭 서브넷 마스크

### 1. 기본 개념
```
서브넷 마스크 비트    사용 가능한 호스트 수    서브넷 수
/24 (255.255.255.0)   254                     1
/25 (255.255.255.128) 126                     2
/26 (255.255.255.192) 62                      4
/27 (255.255.255.224) 30                      8
/28 (255.255.255.240) 14                      16
/29 (255.255.255.248) 6                       32
/30 (255.255.255.252) 2                       64
```

### 2. 서브넷 계산 예시
```
IP 주소: 192.168.123.71
서브넷 마스크: 255.255.255.192 (/26)

네트워크 주소 계산:
192.168.123.71 & 255.255.255.192 = 192.168.123.64

사용 가능한 호스트 범위:
192.168.123.65 ~ 192.168.123.126
```

## 🌐 OSI 7계층

### 1. 계층별 역할
```
7. 응용 계층 (Application)
   - 사용자 인터페이스
   - HTTP, FTP, SMTP

6. 표현 계층 (Presentation)
   - 데이터 형식 변환
   - 암호화/복호화

5. 세션 계층 (Session)
   - 연결 수립/유지/종료
   - 세션 관리

4. 전송 계층 (Transport)
   - 신뢰성 있는 전송
   - TCP, UDP

3. 네트워크 계층 (Network)
   - 경로 설정
   - IP, 라우터

2. 데이터링크 계층 (Data Link)
   - 물리적 주소 지정
   - MAC 주소, 스위치

1. 물리 계층 (Physical)
   - 비트 단위 전송
   - 케이블, 허브
```

## 🔧 네트워크 장비

### 1. 라우터 (Router)
```
- 네트워크 계층(L3) 장비
- 서로 다른 네트워크 연결
- IP 주소 기반 라우팅
- 최적 경로 설정
```

### 2. 스위치 (Switch)
```
- 데이터링크 계층(L2) 장비
- 같은 네트워크 내 통신
- MAC 주소 기반 스위칭
- 충돌 도메인 분리
```

## 📊 시각적 구성도

### 기본 구성
```
[Client A] --- [Switch] --- [Client B]
   |             |            |
192.168.1.2   (Layer 2)   192.168.1.3

설명:
- Switch는 L2 장비로 IP 주소 불필요
- 모든 장비가 같은 네트워크 대역(192.168.1.0/24)
- 각 클라이언트는 고유한 IP 주소 필요
```

### 라우터 구성
```
Network A          Router          Network B
[192.168.1.0/24]    |    [192.168.2.0/24]
      |          [R1|R2]         |
   [Host A]          |        [Host B]

설명:
- R1: 192.168.1.1/24 (Network A의 게이트웨이)
- R2: 192.168.2.1/24 (Network B의 게이트웨이)
- Host A: 192.168.1.x/24, GW: 192.168.1.1
- Host B: 192.168.2.x/24, GW: 192.168.2.1
```

### 서브넷 분할
```
192.168.1.0/24
     |
     |--[0-63]   -> 192.168.1.0/26   (사용 가능: 1-62)
     |--[64-127] -> 192.168.1.64/26  (사용 가능: 65-126)
     |--[128-191]-> 192.168.1.128/26 (사용 가능: 129-190)
     |--[192-255]-> 192.168.1.192/26 (사용 가능: 193-254)

설명:
- 각 서브넷은 62개의 사용 가능한 호스트 주소
- 각 서브넷의 첫 번째(.0)와 마지막(.63) 주소는 예약
- 서브넷 간 통신은 라우터 필요
```

## 💡 레벨별 문제 해결 가이드

### Level 1: 기본 네트워크 이해
```
구성: Client A <-> Client B
목표: 두 클라이언트 간 직접 통신 설정

해결 방법:
1. 같은 네트워크 대역 사용 (예: 192.168.0.x/24)
2. 각 클라이언트에 고유 IP 할당
3. 서브넷 마스크가 동일한지 확인
```

### Level 2: 스위치 네트워크
```
구성: Client A <-> Switch <-> Client B
목표: 스위치를 통한 통신 구성

해결 방법:
1. 모든 장비를 동일 네트워크로 구성
2. 스위치는 L2 장비이므로 IP 설정 불필요
3. 클라이언트들의 IP가 같은 네트워크 대역인지 확인
```

### Level 3: 라우터 기초
```
구성: Network A <-> Router <-> Network B
목표: 서로 다른 네트워크 간 통신

해결 방법:
1. 각 네트워크에 다른 대역 할당 (예: 192.168.0.0/24, 192.168.1.0/24)
2. 라우터 인터페이스에 각 네트워크의 게이트웨이 IP 할당
3. 각 클라이언트의 게이트웨이 설정
```

### Level 4: 인터넷 연결
```
구성: Network <-> Router <-> Internet
목표: 인터넷 연결 구성

해결 방법:
1. 사설 네트워크 대역 사용 (192.168.x.x)
2. 인터넷 연결용 공인 IP 설정
3. 게이트웨이를 통한 외부 접속 설정
```

### Level 5: 서브넷 이해
```
구성: Multiple Networks <-> Router
목표: 서브넷을 이용한 네트워크 분할

해결 방법:
1. 적절한 서브넷 마스크 선택 (/26, /27 등)
2. 각 서브넷의 네트워크 주소 계산
3. 사용 가능한 호스트 IP 범위 확인
```

### Level 6: 복잡한 라우팅
```
구성: Network A <-> Router 1 <-> Router 2 <-> Network B
목표: 다중 라우터 환경 구성

해결 방법:
1. 각 네트워크 대역을 명확히 구분
2. 라우터 간 연결을 위한 IP 설정
3. 각 라우터의 라우팅 테이블 설정
```

### Level 7: 중첩 네트워크
```
구성: Multiple Networks with Overlapping Routes
목표: 효율적인 네트워크 구성

해결 방법:
1. CIDR을 이용한 네트워크 설계
2. 최적의 서브넷 마스크 선택
3. 라우팅 경로 최적화
```

### Level 8: 고급 라우팅
```
구성: Complex Network with Multiple Paths
목표: 다중 경로 네트워크 구성

해결 방법:
1. 각 경로의 네트워크 주소 체계 설정
2. 라우터 간 통신 경로 설정
3. 백업 경로 구성
```

### Level 9: 네트워크 격리
```
구성: Isolated Networks with Controlled Access
목표: 보안을 고려한 네트워크 분리

해결 방법:
1. 각 네트워크 영역 명확히 구분
2. 접근 제어를 위한 IP 설계
3. 게이트웨이를 통한 통신 제어
```

### Level 10: 종합 구성
```
구성: Full Network Implementation
목표: 모든 개념을 통합한 네트워크 구성

해결 방법:
1. 체계적인 IP 주소 할당
2. 효율적인 서브넷 구성
3. 안정적인 라우팅 설정
4. 전체 네트워크 연결성 확인
```

## 🔍 트러블슈팅

### 1. 통신 불가 문제
```
증상: ping 실패
해결:
1. IP 주소 충돌 확인
   - IP 주소 중복 사용 여부 체크
   - 네트워크 대역이 올바른지 확인
2. 서브넷 마스크 일치 확인
   - 같은 네트워크 내 동일한 서브넷 마스크 사용
   - 올바른 CIDR 표기법 사용
3. 게이트웨이 설정 확인
   - 게이트웨이가 해당 네트워크에 존재하는지 확인
   - 게이트웨이 IP가 유효한지 확인
4. 라우팅 테이블 확인
   - 경로가 올바르게 설정되었는지 확인
   - 양방향 통신이 가능한지 확인
```

### 2. 잘못된 네트워크 구성
```
증상: 일부 구간만 통신 가능
해결:
1. 네트워크 대역 중복 확인
   - 각 네트워크가 고유한 대역 사용
   - 서브넷 간 충돌 여부 확인
2. CIDR 표기법 이해
   - 올바른 서브넷 마스크 사용
   - 네트워크 크기 적절성 확인
3. 라우터 인터페이스 설정 검토
   - 각 인터페이스가 올바른 네트워크에 속하는지 확인
   - 인터페이스 IP가 유효한지 확인

증상: 느린 통신 속도
해결:
1. 네트워크 분할 최적화
   - 적절한 크기로 서브넷 분할
   - 브로드캐스트 도메인 크기 조정
2. 라우팅 경로 최적화
   - 불필요한 홉 제거
   - 효율적인 경로 설정
```
